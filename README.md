# lazy-impl
# ⚠️ EXPERIMENTAL - USE AT YOUR OWN RISK ⚠️
This package generates Python code at runtime using LLMs and writes it to your source files.
## Risks
**This is experimental software that modifies your source code automatically.**
- **Generated code quality depends entirely on the LLM** - No guarantees of correctness
- **Code is written directly to your source files** - Can overwrite your work
- **No guarantees of correctness or safety** - Generated code may have bugs or security issues
- **May introduce bugs, security issues, or break your code** - Use with extreme caution
- **Requires API access to LLM services** - Costs money (Anthropic or OpenAI API keys)
- **Runtime code generation is inherently risky** - Not suitable for production use
## Recommended Usage
- ✅ Use only in experimental/prototype projects
- ✅ Always use version control (git) and commit before using
- ✅ Review all generated code before committing
- ✅ Test thoroughly before using in production
- ✅ Consider this a research tool, not a production solution
- ❌ Do NOT use in production systems
- ❌ Do NOT use on code without version control
- ❌ Do NOT use in safety-critical applications
**This is a proof-of-concept for LLM-powered code generation.**

## Quick Start
```python
from lazy_impl import lazy
# Decorate a function with a prompt
@lazy("Implement a quicksort algorithm")
def quicksort(arr: list[int]) -> list[int]:
    pass  # Implementation will be generated by LLM on first call
# Use the function - implementation is generated on first call
result = quicksort([3, 1, 4, 1, 5, 9, 2, 6])
print(result)  # [1, 1, 2, 3, 4, 5, 6, 9]
```
## Features
### `@lazy` - Smart decorator for functions or classes
```python
from lazy_impl import lazy
@lazy("Implement a merge sort algorithm")
def merge_sort(items: list) -> list:
    pass
@lazy("Implement a persistent key-value store with TTL")
class KeyValueStore:
    pass
```
### `@lazy_fn` - Explicit function decorator
```python
from lazy_impl import lazy_fn
@lazy_fn("Sort using bubble sort")
def bubble_sort(arr: list[int]) -> list[int]:
    pass
```
### `@lazy_class` - Explicit class decorator
```python
from lazy_impl import lazy_class
@lazy_class("Implement an LRU cache")
class LRUCache:
    pass
```
## How It Works
1. You decorate a function or class with `@lazy(prompt)`
2. On first call/instantiation, the decorator:
   - Analyzes the function/class signature and docstring
   - Sends a prompt to an LLM (Claude or GPT) with your instructions
   - Generates a complete implementation
   - **Writes the generated code back to your source file**
   - Executes the generated code
3. Subsequent calls use the generated implementation
## Configuration
Set your API key as an environment variable:
```bash
# For Anthropic (Claude)
export ANTHROPIC_API_KEY="your-key-here"
# For OpenAI (GPT)
export OPENAI_API_KEY="your-key-here"
```
## Advanced Usage
### Custom Bot Instance
```python
from lazy_impl import lazy_fn
from bots import AnthropicBot
bot = AnthropicBot(name="CodeGenerator", model="claude-3-5-sonnet-20241022")
@lazy_fn("Implement fibonacci", bot=bot)
def fibonacci(n: int) -> int:
    pass
```
### Context Levels
Control how much context the LLM sees:
```python
@lazy("Implement a parser", context="medium")
def parse_data(text: str) -> dict:
    pass
```
Context levels:
- `"None"`: No additional context (default)
- `"low"`: Only the containing class
- `"medium"`: The entire current file
- `"high"`: Current file and interfaces of other files in directory
- `"very high"`: All Python files in directory
